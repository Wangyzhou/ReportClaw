## 今天这次 session 做了什么（按因果顺序）

### 1. 找到根因
跑 `openclaw agent --agent reportclaw-coordinator` 让它 spawn writer，Coordinator 自己报错：
```
agentId is not allowed for sessions_spawn (allowed: none)
```
追到 `/opt/homebrew/lib/node_modules/openclaw/dist/subagent-spawn-CyV5i9yl.js:448`，发现是 `allowSet` 空。配置路径：`agents.<id>.subagents.allowAgents`（fallback 到 `agents.defaults.subagents.allowAgents`）。

### 2. 改 openclaw.json（per-machine 不入 git）
```json
{
  "id": "reportclaw-coordinator",
  ...
  "subagents": {
    "allowAgents": [
      "reportclaw-retriever",
      "reportclaw-writer",
      "reportclaw-rewriter",
      "reportclaw-reviewer"
    ]
  }
}
```
备份在 `~/.openclaw/openclaw.json.bak.before-subagents-allow-20260505-220848`。新机器上跑 `python3 scripts/setup_openclaw_subagents.py` 一键配（幂等）。

### 3. 同步修 4 个 md（agents/coordinator/）
原 TOOLS.md / SKILL.md 里 agentId 写的是短名 `writer/retriever/reviewer/rewriter`，但实际 OpenClaw 注册的是 `reportclaw-*` 全名 — 即使权限通了，Coord 拿短名调 sessions_spawn 一样会被拒。**全部统一成全名**（在 4 处 .md 各加 ⚠️ 铁律说明，防 LLM 乱回退到短名）。

### 4. 加编排模式（chat-server vs native）
Coord 现在能 spawn 后引发新问题：chat server 调 Coord 时，Coord 自己也会 sessions_spawn，跟 chat server 自己的 retriever/writer/reviewer 调用**撞车 → 双倍 token / 状态竞争**。

修复方式：在用户消息开头注入 `[CHAT-SERVER-MODE]` 指令文，Coord 在 AGENTS.md / SKILL.md 都看到"命中标记 → 严禁 sessions_spawn / write / edit / web_search，唯一输出 fenced dispatch JSON 即结束 turn"。

```
chat server (Python :8081)
  └─ openclaw agent --agent reportclaw-coordinator
       (message prepended with [CHAT-SERVER-MODE] ...)
       → Coord 只输出 ```json ... ```
       → chat server 自己调 retriever/writer/reviewer

CLI 直连
  └─ openclaw agent --agent reportclaw-coordinator
       (message 无标记)
       → Coord sessions_spawn(reportclaw-writer) → announce → ...
       → Coord 多轮回环 (writer / reviewer / writer-revise / reviewer-2)
```

### 5. chat server JSON 抽取改鲁棒
原 regex `r"\{.*?\"intent\".*?\}"` 非贪婪，碰到嵌套 JSON 会在第一个内层 `}` 截断。新版：先找 ```json fenced block（Coord 现在按 SKILL 输出这个），找不到再 balanced-brace stack 扫描，确保抽出完整 dispatch payload。

### 6. 实测两条路径都跑通

| 路径 | 实测时间 | tools used | 结果 |
|------|---------|-----------|------|
| CLI native | 22:13-22:17 (4 轮 turns) | sessions_spawn × 3 + edit + web_search | ✅ 多轮 announce 自动推进 |
| Chat server | 22:30 单查询 | TOOL_CALLS=None on Coord | ✅ Coord 干净返回 dispatch JSON 不 spawn，chat server 走完整流式链 |

## 关键技术决策

| 决策 | 为什么 |
|------|--------|
| 不在 OpenClaw runtime 层加 "advisory mode"（plan-only） | 看了源码 + 文档，OpenClaw 没原生支持。Eddie 没必要 fork OpenClaw，靠 prompt 约定就够 |
| 用 `[CHAT-SERVER-MODE]` 中文指令文而不是 XML 标签 `<orchestration-mode>` | 第一次试 XML 标签，DeepSeek 直接忽略 → 改成"严禁..."明确指令文，秒中 |
| 关键 mode 判定放 AGENTS.md 顶部（不只是 SKILL.md）| AGENTS.md 总在 system prompt 里，SKILL.md 是 lazy load，万一 Coord 不读 SKILL 也得能看到规则 |
| 双路径并存而非二选一 | CLI 直连演示"原教旨多轮编排"卖点；chat-server 路径保前端 UX。两个市场都要 |
| `setup_openclaw_subagents.py` 用 Python 不用 bash | 跨平台（Mac/Linux/WSL2 都跑），JSON 改写比 jq/sed 更稳 |
| `_extract_dispatch_json` 双层兜底 | fenced 优先匹配新规范输出，balanced-brace 兜旧调用 / LLM 偏离格式 — 最坏情况也不会因 regex 截断挂掉 |

## 当前已知边界（demo 不影响，长期改进）

| 项 | 现状 | 不影响什么 / 长期咋办 |
|----|------|-------------------|

## 故障排查

| 症状 | 八成是 | 修法 |
|------|-------|------|
| `agentId is not allowed for sessions_spawn (allowed: none)` | 没跑 setup 脚本 | `python3 scripts/setup_openclaw_subagents.py` |
| 浏览器 502 | chat server 没起 | `bash scripts/start_demo.sh` 或看 `/tmp/chat-server.log` |
| chat server 把 Coord 输出当无效 fallback | dispatch JSON 没匹配上 | 看 chat-server.log，搜 `[coord-openclaw] 无 dispatch JSON` 后面的 text 看 Coord 实际输出长啥样 |
| Coord 在 chat-server 模式还是 sessions_spawn | mode hint 没写对 | grep `_ORCH_MODE_HINT` 看注入字符串，确保 `[CHAT-SERVER-MODE]` 在 message 第一行 |
| OpenClaw 进程僵尸 | 之前 kill -9 没清干净 | `ps aux | grep openclaw` + `kill -9 <pid>` + 重启 daemon |
| Spring Boot 端口冲突 | 老 demo 没 stop | `lsof -i :8080` + `kill <pid>` |
