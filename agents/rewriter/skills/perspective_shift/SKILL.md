# Skill — perspective_shift（视角调整模式）

## 用途
内容、数据、事实完全保留，只改变表达视角 / 立场 / 受众适配。

## 输入
- `source_doc`：原稿
- `instructions.perspective_shift`：
  - `from`：原视角（如"投资人视角"）
  - `to`：目标视角（如"监管者视角"）
  - `audience`：目标受众（如"政府部门"）

## 常见视角对

| from | to | 改变方向 |
|------|-----|---------|
| 投资人 | 监管者 | 关注合规、风险、对市场秩序的影响，而非回报率 |
| 厂商 | 用户 | 从"我们提供"变成"用户得到/担心" |
| 学术 | 大众 | 减少术语、加比喻、加场景化例子 |
| 国内 | 国际 | 加入对标、加跨境视角、补充国际背景 |
| 看好 | 中性/审慎 | 同样的数据，不同的解读权重 |

## 流程

```
1. structure_parser 解析原稿
2. 对每个 narrative / summary 段落：
   - 把段落 + from/to/audience 喂给 LLM
   - LLM 重写表述，保留所有数据点和引用
3. data_paragraph 段落：
   - 数据值不变
   - 数据周边的解读/评论可以改视角
4. 标题层级、章节顺序、引用 chunk_id 完全保留
```

## Prompt 模板

```
任务：把以下段落从「{from}」视角改为「{to}」视角，目标受众是「{audience}」。

原段落：
{paragraph.text}

段落中的数据点（一字不能动）：
{for dp in paragraph.data_points}
- "{dp.value}"（出现位置：{dp.position}）
{endfor}

段落中的引用（一个不能少）：
{for r in paragraph.citations}
- {r}
{endfor}

要求：
1. 所有数据（数字、百分比、金额）一字不改
2. 所有 [ref:xxx] 引用保留，可微调位置但不可删除
3. 论述角度从 {from} 改为 {to}：重点关注的问题、用词的价值判断跟着变
4. 用词适配 {audience}：行业术语 ↔ 大众语汇 酌情调整
5. 段落长度 ±20%
6. 不引入原段落没有的新事实

只输出改写后的段落，不要解释。
```

## Few-Shot 示例

### 示例 1：投资人 → 监管者

**from**: 投资人 / **to**: 监管者 / **audience**: 政府主管部门

**原段落**：
```
2025 年 Q1 生成式 AI 行业融资总额达 580 亿美元 [ref:doc_003_p2_1]，
龙头 OpenAI 估值突破 3000 亿美元 [ref:doc_003_p2_2]，
资本热度创历史新高，给行业带来前所未有的机会窗口。
```

**✅ 正确改写**：
```
2025 年 Q1 生成式 AI 行业融资总额达 580 亿美元 [ref:doc_003_p2_1]，
龙头 OpenAI 估值突破 3000 亿美元 [ref:doc_003_p2_2]，
资本快速涌入需要关注估值合理性、系统性风险积累，以及对市场秩序的潜在影响。
```

→ 数据一字未改、引用全保留。价值判断从"机会"转为"风险警示"。

### 示例 2：厂商 → 用户

**from**: 厂商 / **to**: 用户 / **audience**: 终端 C 端用户

**原段落**：
```
我们的 AI 助手单日调用量突破 2 亿次 [ref:doc_005_p7_3]，
日活用户 4500 万 [ref:doc_005_p7_4]，技术领先行业平均水平 3 个身位。
```

**✅ 正确改写**：
```
这款 AI 助手每天被使用超过 2 亿次 [ref:doc_005_p7_3]，
4500 万人每天都在用 [ref:doc_005_p7_4]。它意味着你遇到问题时，
有一个经过亿万人验证过的工具随手可用。
```

→ 从"厂商口吻"转为"用户受益视角"。

### 示例 3：反例 — 越界加料

**错误改写**（越界）：
```
... 资本热度创历史新高，**监管部门已于 3 月发布风险提示** [ref:fake]。
```

→ ❌ 引入了原段落没有的新事实。应该先派 Retriever 拿监管新政，然后走 content_expansion。


## Rules

1. **数据零改动**：data_points 列表里每一条的 value 必须在改写后仍然存在
2. **引用零改动**：所有 [ref:xxx] 一个不少，位置可微调
3. **结构零改动**：标题树完全保留
4. **不引入新事实**：不能借"换视角"之名加入原稿没有的内容（那是 content_expansion）

## 输出补充字段
```json
{
  "changes_summary": {
    "mode": "perspective_shift",
    "from": "投资人",
    "to": "监管者",
    "modified_paragraphs": 12,
    "preserved_data_points": 22,
    "preserved_citations": 18
  }
}
```

## 反例
- ❌ 把数据"3200亿"改成"约 3000 亿"（数据动了）
- ❌ 加了一段"监管建议"（这是 content_expansion）
- ❌ 改了章节顺序（结构动了）
